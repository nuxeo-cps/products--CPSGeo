Constructor Detail

ModelBase

ModelBase(model, modelNode, parentModel)
    Base Model class to be inherited by all Model objects and provdes methods and properties common to all models. Stores the XML document as the .doc property of the model. Inherits from the Listener class so all models are also listener objects that can call registered listeners.
    Parameters:
      model - Pointer to the model instance being created
      modelNode - The model's XML object node from the configuration document.
      parentModel - The model object that this model belongs to.
    Author: Cameron Shorter

createObject

Object createObject(configNode)
    Creates all mapbuilder JavaScript objects based on the Object nodes defined in the configuration file. A reference to the created model is stored as a property of the config.objects property using the model's ID; you can always get a reference to a mapbuilder object as: "config.objects.objectId"
    Parameters:
      configNode - The node from config for the model to be created

loadModelDoc

void loadModelDoc(modelRef)
    Load a Model's document. This will only occur if the model.url property is set. Calling this method triggers several events: modelStatus - to indicate that the model state is changing newModel - to give widgetrs a chance to clear themselves before the doc is loaded loadModel - to indicate that the document is loaded successfully refresh - to indicate that widgets should be refreshed
    Parameters:
      modelRef - Pointer to the model object being loaded.

newRequest

void newRequest(modelRef, httpPayload)
    Load XML for a model from an httpPayload object To update model data, use:
    httpPayload=new Object();
    httpPayload.url="url" or null. If set to null, all dependant widgets will be removed from the display.
    httpPayload.httpMethod="post" or "get"
    httpPayload.postData=XML or null
    Parameters:
      modelRef - Pointer to the model object being loaded.
      httpPayload - an object to fully specify the request to be made
